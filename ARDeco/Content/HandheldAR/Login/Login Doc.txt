Documentation du Processus de Connexion et d'Inscription
Objectif
Ce document explique le processus de connexion et d'inscription dans l'application en utilisant deux champs de texte pour le courriel (email) et le mot de passe (mdp), ainsi que des boutons de connexion et d'inscription.

Étapes du Processus
1. Interface Utilisateur
Champs de Texte :

L'application propose deux champs de texte, l'un pour le courriel et l'autre pour le mot de passe.
Bouton de Connexion :

L'utilisateur peut appuyer sur un bouton de connexion pour soumettre ses informations.
Bouton d'Inscription :

Un bouton "Register" est disponible pour permettre aux utilisateurs de s'inscrire.
2. Récupération des Données pour Connexion
Bouton de Connexion Pressé :

Lorsque l'utilisateur appuie sur le bouton de connexion, le système récupère les données saisies dans les champs de texte.

3. Construction de la Requête pour Connexion
Construction de la Chaîne JSON :

Les données récupérées sont combinées pour former une chaîne JSON utilisant le nœud Make Literal String.

Création de la Requête HTTP :

Utilisation du nœud Make HTTP Request pour créer une requête POST avec la chaîne JSON en tant que corps de la requête.
4. Envoi de la Requête et Traitement de la Réponse pour Connexion
Envoi de la Requête :

La requête HTTP est envoyée au serveur.
Traitement de la Réponse :

Le corps de la réponse est extrait et converti en JSON.

Extraction du Code de Réponse :

Le nœud Get Field est utilisé pour extraire le code de réponse du JSON.

Vérification du Code de Réponse :

Utilisation d'une structure conditionnelle pour vérifier si le code de réponse est 200 (OK) ou 401 (Non autorisé).

5. Redirection vers le HUD d'Inscription
Bouton "Register" Pressé :
Lorsque l'utilisateur appuie sur le bouton "Register", le système redirige l'utilisateur vers le HUD d'inscription.
